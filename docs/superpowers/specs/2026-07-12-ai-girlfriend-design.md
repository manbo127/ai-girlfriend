# AI 女友网页 — 设计文档

**日期**: 2026-07-12  
**状态**: 已确认

---

## 1. 项目概述

一个聊天式 AI 女友网页应用，核心能力：记住用户性格爱好、使用性格模板、主动关心用户。

## 2. 技术选型

| 维度 | 方案 |
|------|------|
| 前端 | 纯 HTML + CSS + JS，无框架，单页应用 |
| 部署 | GitHub Pages |
| 版本控制 | Git + GitHub |
| AI API | DeepSeek API（OpenAI 兼容格式 + Function Calling） |
| 本地存储 | IndexedDB（聊天记录、记忆）+ localStorage（设置） |
| 后续扩展 | WebSocket 连接本地 Python 服务实现桌面操作 |

> **安全提醒**：API Key 存储在浏览器 localStorage 中，不会提交到 Git。使用 `.gitignore` 确保任何含 Key 的配置文件不被追踪。

## 3. 系统架构

```
浏览器 (网页)
├── 聊天界面 (主界面)
├── 记忆管理 (关于你)
├── 性格模板管理
├── AI 对话引擎
│   ├── 构建 System Prompt (模板 + 记忆 + 时间)
│   ├── 调用 DeepSeek API
│   └── 解析回复 & 提取记忆
└── 本地存储层 (IndexedDB)
    └── 聊天记录 / 记忆数据 / 模板 / 设置

后续扩展:
└── WebSocket → 本地 Python 服务 → 工具执行 / 桌面自动化
```

## 4. 聊天界面

- 顶栏：当前模板名，可点击切换
- 消息区：她的消息左侧粉色气泡，用户消息右侧蓝色气泡
- 她回复时显示打字动画
- 系统记忆提取轻量提示
- 底部输入区：文本输入 + Enter 发送

## 5. 性格模板系统

- **预设模板**：仅「傲娇毒舌型」一个预设
- **自定义模板**：用户可新建、编辑、删除自己的模板
- 每个模板是一段 System Prompt 文本，定义性格、语气、称呼、关心方式
- 存储：IndexedDB

## 6. 记忆系统

### 手动填写
- 称呼、生日、性格、爱好、讨厌、其他
- 直接拼入 System Prompt

### 自动提取
- 每轮对话后，AI 判断是否有值得记住的新信息
- 提取后存入列表，带时间戳
- 上限 50 条，旧的自动清理
- 用户可手动删除单条或清空
- 对话时带上最近相关记忆

## 7. 主动问候

- 打开网页时根据时间段自动发送问候（早上/中午/下午/晚上/深夜各不同）
- 浏览器通知为可选功能，默认关闭
- 开启后可设置固定时间提醒

## 8. 工具调用 & 电脑操作

### v1.x — Function Calling 工具
- 查天气、搜索、设提醒、时间日期
- 前端直接执行工具调用

### v2.0 — 本地服务 + 电脑操作
- WebSocket 连接本地 Python 服务
- 文件操作、打开软件、键盘输入、鼠标控制、截图
- 所有操作需用户确认

## 9. 开发阶段

| 版本 | 内容 |
|------|------|
| v1.0 | 聊天 + 模板 + 记忆 + 问候 |
| v1.1 | 工具调用（天气、搜索、提醒） |
| v2.0 | 本地服务 + 电脑操作 |

## 10. 数据安全

- 所有数据存于用户浏览器 IndexedDB
- API Key 存于 localStorage，仅在前端使用
- 对话数据不经过任何第三方服务器（除 DeepSeek API）
- 无后端，无数据库，无账号系统
