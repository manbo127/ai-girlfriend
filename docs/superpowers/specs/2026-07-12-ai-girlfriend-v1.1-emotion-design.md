# AI 女友 v1.1 — 情感深度 & 主动性增强

**状态**: 已确认

## 1. 概述

在 v1.0 基础上增加两个核心系统：情绪引擎和主动行为系统，让 AI 女友更有"真人感"。

## 2. 新增文件

```
js/mood.js        # 情绪状态机
js/proactive.js   # 主动行为系统
```

## 3. 情绪系统 (mood.js)

### 3.1 情绪模型

四维情绪向量，每个维度 0-100：

| 维度 | 英文 key | 说明 |
|------|---------|------|
| 愉悦度 | `happy` | 受正面互动提升，负面事件降低 |
| 亲密度 | `closeness` | 累积值，随聊天次数和时间缓慢增长 |
| 撒娇度 | `pouty` | 随空闲时间增长，互动后回落 |
| 担忧度 | `worried` | 用户说负面事件时升高，安慰后降低 |

### 3.2 变化规则

**对话触发（每次用户发消息或 AI 回复后）：**
- 正面内容（夸奖、感谢、分享开心事）→ happy +5, closeness +3
- 负面内容（抱怨、难过、压力）→ worried +10, happy -5
- 普通闲聊 → happy +2
- 吵架/矛盾 → happy -15, pouty +20

**时间触发（定时器检查）：**
- 每闲置 30 分钟 → pouty +10
- 深夜（0-6点）且用户在线 → worried +5
- 每小时自然衰减：happy -1, pouty +2, worried -1（向基线回归）

**上限保护：** 所有值限制在 [0, 100]

### 3.3 情绪注入 Prompt

每次构建 System Prompt 时，根据情绪值生成一段"情绪状态描述"：

- happy > 70 → "你现在心情很好，语气轻快活泼"
- happy < 30 → "你现在有点低落，语气会比平时更尖锐"
- pouty > 60 → "你现在很在意他刚才对你不够热情，用闹别扭的方式表达"
- worried > 50 → "你现在有些担心他，忍不住想唠叨他几句"
- closeness > 80 → "你已经很信任和依赖他了，语气中会不自觉地流露"

情绪描述拼接在模板内容之后、记忆信息之前。

### 3.4 存储

情绪状态存入 IndexedDB，页面刷新后保持。

## 4. 主动系统 (proactive.js)

### 4.1 空闲触发

页面打开时启动空闲计时器。用户发消息后重置。

| 空闲时长 | 行为 |
|---------|------|
| 3 分钟 | 发送 "..." （只有省略号，不算正式消息） |
| 10 分钟 | 发送一条试探消息 |
| 30 分钟 | 发送一条抱怨/撒娇消息 |
| 之后 | 不再重复触发 |

具体消息内容由 AI 根据当前情绪状态生成（调用一次 API）。

### 4.2 时间节点

页面打开时，如果当前时间经过以下节点，1 分钟内自动发送：

| 时间段 | 消息 |
|--------|------|
| 11:30-12:30 | 提醒吃午饭 |
| 18:30-19:30 | 问晚饭 |
| 22:30-23:30 | 催睡觉 |

如果用户在这个时间段刚刚发过消息，跳过。

### 4.3 话题回顾

每次聊天最后一个用户消息，由 AI 判断是否包含"待跟进话题"（pending topic）。如果有，存入 IndexedDB。

下次打开网页时（新的会话），如果有待跟进话题，自动发送追问：

> "对了，上次你不是说[话题]吗，后来怎么样了？"

追问只能一次，发出后清空待跟进话题。

### 4.4 节奏控制

- 每次 AI 回复后至少间隔 30 秒才能再发主动消息
- 同一次会话中，同类型主动消息不重复
- 用户在 1 分钟内刚发过消息→所有主动消息暂停

| 文件 | 改动 |
|------|------|
| `js/ai.js` | `buildSystemPrompt()` 增加情绪状态注入 |
| `js/chat.js` | `handleSend()` 后更新情绪状态、检测待跟进话题；增加空闲计时器重置 |
| `js/app.js` | `init()` 中初始化 mood 和 proactive 模块 |
| `js/storage.js` | 新增 mood 和 pendingTopic 存储 |

## 6. 全局约束

- 主动消息仅页面打开时触发，不使用 Service Worker 或后台脚本
- 情绪状态存 IndexedDB，与 v1.0 数据安全原则一致
- 主动消息 API 调用量控制：每 10 分钟最多 1 次额外 API 调用
